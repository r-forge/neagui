neaGUI

Network Enrichment Analysis GUI
Installation

To install the package, perform the following command:

> install.packages("neaGUI")
> library(neaGUI)
> neaGUI()


neaGUI Main Window

After the package installed and the newGUI() function is run, the main window of neaGUI package as shown in the snapshot below will pop up.





























The following options need to be specified:

  1. Altered gene set (AGS).The main input of neaGUI is a list of altered gene sets (AGSs) which is usually a list of differentially expressed genes. Note that the imputed AGS should be ranked in advance based on user defined criteria (e.g., p-values). Example of AGS input can be found here.

  1. The statistics of the NEA method , fixed NEA (FNEA, default) and maximum NEA (MNEA). FNEA depends on the size of ags while MNEA avoids the dependence by taking the largest statistic across AGS. More detail please see Alexeyenko et al. (2012).

  1. Functional gene sets (FGS). Options to specify  known gene sets, databases of group of genes describing their biological activities are necessary. The options provided are GO ontologies (cellular component, biological process, moelcular function) or KEGG pathway. Users can provide their own functional gene sets as a list. Please see the format of AGS input.

  1. Annotation database. The annotation database options which will provide detailed information about the pathways. Besides two provided annotation database options, KEGG pathways(based on KEGG.db package) and Gene Ontologies (based on GO.db), user defined or other annotation database can be specified.

  1. Gene network links.  The network input is a vector of gene pairs (two gene symbols with separation) or a list representing the network link. Users are recommended to use a comprehensive merged gene networks discussed by Alexeyenko et al. (2012) which contains 1,445,027 links of 16,299 distinct Human Proteome Organisation (HUPO) genes. The network can be downloaded from http://www.meb.ki.se/~yudpaw. Example of network input can be found here.

  1. Number of permutation. The default number of permutation is 50, however user can increase the number. Note that this permutation procedure only needs to be performed once for each network list. Then the result of permutated network can be used for other analysis for the same network list using options "load previously saved permutations". A permutated network for the merged network using 200 permutations can be obtained from http://www.meb.ki.se/~yudpaw.

  1. Name of the output. Note that the result files will be saved in the same directory of the AGS input directory.
